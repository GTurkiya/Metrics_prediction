# Прогнозирование пользовательской активности

### Описание проекта
Последнее время в службу поддержки стали чаще приходить жалобы о том, что приложение подвисает. Руководство приняло решение о возможном подключении дополнительных серверов. Цель данного проекта - спрогнозировать пользовательскую активность на ближайшее время, чтобы оптимизировать расход вычислительных мощностей.

### Стек
Python, Pandas, Orbit, Matplotlib, SQL

### Выполненные задачи
1. Определил ключевую метрику — количество просмотров (почасовые данные за 2,5 месяца)
2. Определил оптимальный горизонт прогнозирования (на неделю вперед)
3. Выгрузил данные с помощью SQL, сформировал временной ряд
4. Обработал аномалию, связанную с проведением флешмоба

<img width="1241" height="382" alt="image" src="https://github.com/user-attachments/assets/53b25cf8-edef-44a8-9c18-55abaffc6460" />

5. Подобрал подходящий регрессор (количество активных пользователей на каждый час), тем самым, увеличив точность прогнозирования
6. Построил несколько алгоритмов прогнозирования на основе исторических данных, провалидировал их с помощью BackTester и выбрал лучший
7. Выполнил прогнозирование с 90%-м доверительным интервалом и интерпретировал результаты

### Основные результаты
1. Было получено значение метрики MAPE, примерно равное 0,12, и рассчитан 90%-й доверительный интервал
